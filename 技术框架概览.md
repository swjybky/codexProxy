# Codex Proxy Desktop 技术框架概览

> 本项目沿用“Python 桌面壳 + React SPA + 本地 HTTP 服务 + 本地 JSON 存储”的桌面架构，将业务收敛为 Codex Responses 专用反向代理。详细协议见 [`项目整体设计.md`](./项目整体设计.md)。

## 1. 总体架构

```text
┌────────────────────────────────────────────────────┐
│  pywebview 桌面窗口                                 │
│  React 19 + TypeScript + Vite 管理界面              │
└──────────────────────┬─────────────────────────────┘
                       │ 同源管理 API
┌──────────────────────▼─────────────────────────────┐
│  Python ThreadingHTTPServer（0.0.0.0:1455）         │
│  ├─ /api/*：状态、凭证、设置                        │
│  ├─ /v1/responses                                   │
│  └─ /v1/responses/compact                           │
└──────────────────────┬─────────────────────────────┘
                       │ OAuth + Codex 专用 Header
┌──────────────────────▼─────────────────────────────┐
│  chatgpt.com/backend-api/codex/responses            │
│  JSON / SSE                                         │
└────────────────────────────────────────────────────┘
```

## 2. 技术选型

| 类别 | 技术 | 说明 |
|---|---|---|
| 后端 | Python 3.10+ 标准库 | HTTP、JSON、OAuth 刷新、流式转发 |
| 桌面壳 | pywebview 5.x | Windows WebView2 / macOS WKWebView / Linux Qt 或 GTK |
| 前端 | React 19 + TypeScript + Vite | 简约单页管理界面 |
| 存储 | 本地 JSON + 原子写入 | 设置与凭证分文件保存，不使用数据库 |
| 打包 | PyInstaller onedir / macOS app bundle | 前端 `web/dist` 与 Windows `.ico`、macOS `.icns`、Linux PNG 图标随桌面程序分发 |

## 3. 核心模块

| 路径 | 职责 |
|---|---|
| `app/main.py` | 进程入口、HTTP 服务与桌面窗口生命周期 |
| `app/server.py` | 本地路由、Bearer 鉴权、静态资源与 SSE 透传 |
| `app/codex_proxy.py` | Codex 请求转换、URL/Header 适配、凭证刷新 |
| `app/storage.py` | 设置、OAuth 凭证、Codex CLI 凭证导入 |
| `app/runtime_paths.py` | 跨平台资源与用户数据路径 |
| `web/src/App.tsx` | 运行、接入、凭证、代理设置界面 |

## 4. 运行与打包

```bash
cd web && npm install && npm run build && cd ..
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/python run.py
```

Windows 一键打包：

```powershell
.\packaging\build_windows.ps1
```

## 5. 设计原则

- 仅处理 Codex Responses 协议，不扩展成多渠道网关。
- 同时监听本机与局域网；代理接口需要 API Key，管理接口另需桌面管理令牌。
- 上游 OAuth 与客户端本地 API Key 隔离。
- 请求前自动刷新即将过期的 OAuth 凭证，上游 401 时刷新后仅重试一次。
- 请求转换与 new-api `relay/channel/codex/adaptor.go` 保持同一语义。
- 普通 JSON 与 SSE 都不做业务内容改写，降低协议漂移风险。
